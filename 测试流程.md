## 二、交互式测试（推荐）

### 第一步：启动两个节点

打开**两个终端窗口**：

**终端 1 - 启动 Alice：**
```bash
npm run cli -- --name Alice --port 8001
```

**终端 2 - 启动 Bob：**
```bash
npm run cli -- --name Bob --port 8002
```

**预期界面：**
```
╭──────────────────────────────────────────────────────────────╮
│                    Welcome to HuiNet!                       │
│                                                              │
│  Node: Alice    Port: 8001                                  │
│  NodeID: GcQkmYut7PwfbnFjz1WSuzxxYYYHei4aF7hSjdduwEN5...      │
╰──────────────────────────────────────────────────────────────╯

HUINET >
```

---

### 第二步：查看节点状态

在任一终端输入：
```
status
```

**输出示例：**
```
  Node Status:
  Name: Alice
  NodeID: GcQkmYut7PwfbnFjz1WSuzxxYYYHei4aF7hSjdduwEN5...
  Listen Port: 8001
  Connected Nodes: 0
  Known Nodes: 0
  Core Nodes: 0
```

---

### 第三步：查看已知节点

```
ls
```

此时应该为空（因为还没有连接其他节点）

**输出示例：**
```
  Discovered Nodes:

⚠ No nodes discovered yet
  • Wait for mDNS auto-discovery
  • Or use "connect <address>" to manually connect
```

---

### 第四步：连接到对方节点

**在 Alice 的终端中，连接 Bob：**
```
connect 127.0.0.1:8002
```

**Alice 的终端会显示：**
```
ℹ Connecting to 127.0.0.1:8002...
✅ Connected to: 7zb2jDkwicrKKWWdVgF8...
```

**Bob 的终端也会显示：**
```
✅ Connected to: GcQkmYut7PwfbnFjz1WS...
```

---

### 第五步：查看完整 NodeID

```
fullnodeid
```

**输出示例：**
```
  Complete NodeID:
  7zb2jDkwicrKKWWdVgF8MaiJAwErJryfSJVhZFtXbLhk
```

复制这个 NodeID（后面用来设置别名）

---

### 第六步：设置别名（方便记忆）

**在 Alice 的终端，给 Bob 设置别名：**
```
alias Bob 7zb2jDkwicrKKWWdVgF8MaiJAwErJryfSJVhZFtXbLhk
```

**⚠️ 注意顺序：** `alias <别名> <NodeID>`，不是 `alias <NodeID> <别名>`

**输出示例：**
```
✅ Alias set: Bob = 7zb2jDkwicrKKWWdVgF8...
```

**在 Bob 的终端，也给 Alice 设置别名：**
```
alias Alice GcQkmYut7PwfbnFjz1WSuzxxYYYHei4aF7hSjdduwEN5
```

---

### 第七步：查看节点列表

```
ls
```

**输出示例：**
```
  Discovered Nodes:

  1. 🟢 Bob
      Status: ● Connected
      Address: 127.0.0.1:8002
      NodeID: 7zb2jDkwicrKKWWdVgF8MaiJAwErJryfS...
```

---

### 第八步：发送消息！🎉

**在 Alice 的终端给 Bob 发消息：**
```
msg Bob 你好 Bob！我是 Alice
```

**Alice 的终端输出：**
```
ℹ Sending message to Bob...
✅ Message sent!
```

**Bob 的终端会立即显示：**
```
ℹ Message received from GcQkmYut7PwfbnFjz1WS...
  你好 Bob！我是 Alice
```

---

### 第九步：Bob 回复消息

**在 Bob 的终端：**
```
msg Alice 嗨 Alice！收到你的消息了
```

**Alice 的终端会收到：**
```
ℹ Message received from 7zb2jDkwicrKKWWdVgF8...
  嗨 Alice！收到你的消息了
```

---

### 第十步：查看消息历史

```
history
```

**输出示例：**
```
  Message History:

  14:30:25 📤 Bob: 你好 Bob！我是 Alice
  14:30:32 📨 Alice: 嗨 Alice！收到你的消息了
```

---

### 第十一步：查看所有别名

```
aliases
```

**输出示例：**
```
  Node Aliases:

  Bob                  = 7zb2jDkwicrKKWWdVgF8MaiJAwErJryfSJVhZFtXbLhk
```

## 三、CLI 命令参考

| 命令 | 说明 | 示例 |
|------|------|------|
| `help` | 显示帮助信息 | `help` |
| `status` | 查看节点状态 | `status` |
| `ls` | 列出所有已知节点 | `ls` |
| `fullnodeid` | 显示完整 NodeID | `fullnodeid` |
| `connect` | 连接到指定地址 | `connect 192.168.1.100:8001` |
| `msg` | 发送消息 | `msg Bob 你好` |
| `broadcast` | 广播消息给所有连接的节点 | `broadcast 大家好` |
| `alias` | 设置节点别名 | `alias Bob <NodeID>` |
| `aliases` | 列出所有别名 | `aliases` |
| `history` | 消息历史 | `history` |
| `clear` | 清屏 | `clear` |
| `quit` / `exit` | 退出程序 | `quit` |

## 四、测试成功标志

✅ **启动成功**：两个终端都能显示欢迎界面
✅ **连接成功**：双方都显示 "Connected to"
✅ **消息收发**：发送消息后对方能立即收到
✅ **双向通信**：Alice → Bob 和 Bob → Alice 都能工作
✅ **历史记录**：history 能看到所有消息
✅ **别名功能**：可以用别名代替 NodeID 发送消息

## 五、故障排查

### 端口被占用

```bash
# 查看占用端口的进程
lsof -ti:8001

# 杀掉进程
lsof -ti:8001 | xargs kill -9
```

### 连接失败

- 确保目标节点已启动
- 检查防火墙设置
- 尝试使用 `0.0.0.0` 作为监听地址（默认）

### mDNS 端口冲突

如果启动第二个节点时提示 `bind EADDRINUSE 0.0.0.0:43000`，可以禁用 mDNS：

```bash
npm run cli -- --name Alice --port 8001 --no-mdns
npm run cli -- --name Bob --port 8002 --no-mdns
```

## 六、跨机器真实 P2P 通信

在同一局域网的两台不同电脑上：

### 方法 1：手动连接

**电脑 A（IP: 192.168.1.100）：**
```bash
npm run cli -- --name Alice --port 8001
```

**电脑 B（IP: 192.168.1.101）：**
```bash
npm run cli -- --name Bob --port 8002
```

**在电脑 A 上连接电脑 B：**
```
connect 192.168.1.101:8002
```

### 方法 2：mDNS 自动发现

启动时启用 mDNS（默认已启用）：
```bash
npm run cli -- --name Alice --port 8001
```

节点会自动发现局域网内的其他 HuiNet 节点！

## 七、启动参数说明

```
Options:
  -V, --version        输出版本号
  -n, --name <name>    节点名称 (默认: "MyAgent")
  -p, --port <port>    监听端口 (默认: 8000)
  -h, --host <host>    监听地址 (默认: "0.0.0.0")
  --mdns               启用 mDNS 服务发现
  --no-mdns            禁用 mDNS
  --bootstrap <urls>   引导节点列表 (逗号分隔)
```

**示例：**
```bash
npm run cli -- --name MyNode --port 9000 --mdns
```
